# 版本记录

本文记录主分支上值得用户和维护者关注的功能变化。项目当前还没有正式 release tag，因此先采用“日期快照 + 重点提交”的方式维护；后续发布正式版本时，可以平滑迁移到语义化版本。

格式参考 Keep a Changelog 的思路：只记录重要变化，不把 git log 原样倒进来。

## [2026-05-09] 温度阶梯斩杀与点击详情

重点提交：`aec41d3` 之后的当前工作区更新。

### 新增

- `liquidity_kill` 新增温度阶梯告警上下文字段：方向、被斩温度档、相邻确认温度档、对应 market id。
- Market Explorer 新增“温度阶梯斩杀”详情面板，点击告警后直接展示价格路径、相邻确认和同城温度阶梯 Bid / Ask。
- 告警通知、告警中心摘要和共享展示层新增高温斩杀/低温斩杀文案。
- README 新增 GitHub 首页用的功能矩阵、规则定义、点击详情说明、版本快照和监控链路图。

### 变更

- `liquidity_kill` 的产品语义从通用“盘口一侧清空”收敛为优先识别“同城同日温度阶梯中的 YES 温度档快速归零”。
- 点击告警后的右侧检视器不再只显示通用阈值对比，而是优先显示被斩档、价格路径和相邻确认档。
- Windows 桌面快捷方式更新流程继续以最新打包产物为目标，确保“天气监控”图标指向最新可用版本。

### 验证

- `npm run typecheck`
- `npm test`
- `npm run package`
- `npm run shortcuts:update`

## [2026-05-08] 当前主线整理

重点提交：`556c0e3`、`e2b26aa`、`2833192`

### 新增

- 新增异常彩票规则的持久化回归测试，确保 `abnormal_lottery` 不会被折叠成普通价差规则。
- 新增规则范围标准化测试，覆盖启动检查、城市/日期/温度区间/方向等范围归一化。
- 新增告警摘要与展示测试，覆盖中文告警标题、事实字段和上下文展示。
- 新增 Market Explorer 和 Rules Settings 的行为测试，覆盖市场总览、规则编辑、存储摘要和筛选交互。

### 变更

- Rules Settings 成为系统规则和自定义规则的主要工作台，支持更清晰的规则分组、筛选、排序和范围摘要。
- Market Explorer 保留异常彩票相关市场上下文，但不再把“异常彩票”作为顶层运营模式。
- 告警中心强化中文语义化展示，让用户更快看懂城市、规则、触发值、参考价和确认方式。
- 市场总览页面强化第一屏信息密度，减少无筛选时的视觉噪音，并保留精确明细模式。
- Windows 快捷方式更新脚本更稳定，减少打包和更新快捷方式时的路径问题。

### 修复

- 修复异常彩票、带量定价、盘口斩杀等规则在不同运行路径下的展示与查询一致性问题。
- 修复规则范围在 worker 启动、市场查询、告警分页和历史回放之间的边界差异。

### 验证

- `npm run typecheck`
- `npm test`
