# 市场总览 / 市场探索重构设计

## 本步目标

这一轮只完成设计，不进入实现。

目标是把当前“筛选栏 + 表格”的市场探索页，重构为更适合监控和决策的可视化工作台，同时保留筛选、排序、刷新这些必要操作。

## 现状问题

当前页面的核心问题不是“数据不够”，而是“信息层级不对”：

- 所有市场都被压成同一层级的表格行，用户只能逐行扫读。
- 价格、价差、波动、风险、更新时间没有形成明显的视觉优先级。
- 从首页泡泡双击进入探索页后，体验会突然从“可视化监控”断裂成“密集表格录入感”。
- 筛选控件和结果区之间关系很弱，用户不容易快速知道“我现在筛到了什么”。
- 表格更适合精确核对，不适合作为主监控界面。

## 设计前提

- 保留现有项目的深色监控台风格，不另起一套视觉语言。
- 保留现有查询能力作为第一阶段边界：`cityKey`、`eventDate`、`watchlistedOnly`、`sortBy`、`sortDir`。
- 不先改后端协议，优先用现有 `MarketRow` 字段完成第一版可视化重构。
- 表格不彻底删除，但降为次级模式或后备模式，不再作为默认主视图。

## Frontend Skill 设计骨架

### Visual Thesis

这不是一张“数据表”，而是一块能快速扫到异常、快速定位城市、快速进入单个市场细节的监控工作台。

### Content Plan

- 主工作区：市场可视化总览
- 支撑区：快速筛选和结果摘要
- 细节区：选中市场检视器
- 后备区：精确明细模式

### Interaction Thesis

- 筛选变化时，主视图做平滑重排，而不是整页生硬刷新。
- 选中某个市场后，右侧检视器即时展开，保持上下文不跳走。
- 风险、波动、价差通过颜色、密度、徽标三层同时表达，减少纯文字阅读成本。

## 重构后的页面定位

页面建议统一命名为“市场总览”，副标题强调“筛选 + 可视化定位 + 单市场检视”。

它应该是：

- 首页泡泡之后的“二级工作台”
- 用户做筛选、定位、比对、判断的主页面
- 告警中心和规则管理之间的桥梁页

它不应该是：

- 纯数据导出页
- 只有表格的后台管理页
- 视觉上比首页更弱的一页

## 默认主视图：市场带状总览

默认不再直接展示表格，改成“按城市分组的市场带状视图”。

每个城市是一个分组区块，每个市场是一块可点击的“市场带”：

- 主标题：温度区间
- 副标题：方向、机场、更新时间
- 大数字：`“是”价格`
- 辅助值：`价差`、`5 分钟变化`
- 状态表达：是否关注、是否高波动、是否停牌/已结算

### 为什么用“带状视图”而不是继续上表格

- 比表格更适合横向比较同一城市下多个市场。
- 比卡片墙更节制，密度更高，不会变成一页碎卡片。
- 和首页泡泡页有连续性：先看城市，再看城市下的市场。
- 现有字段足够支撑，不需要先等新接口。

## 市场带的视觉编码

每个市场带同时承担“展示数据”和“表达风险”的作用。

- 底色强度：使用 `bubbleSeverity`
  - `critical`：偏红
  - `warning`：偏橙
  - `info`：偏蓝
  - `none`：偏灰蓝
- 左侧强调条：表达当前风险级别，便于扫读。
- 价格数字：作为视觉主焦点，字号最大。
- 价差：高于阈值时边框加强、文字转暖色。
- 5 分钟变化：正负分别使用上升色 / 下降色。
- 更新时间：用“新鲜度标签”表达，例如“刚更新 / 2 分钟前 / 数据偏旧”。
- 关注盘口：用星标或胶囊徽标表达，不混进正文。

## 右侧检视器

右侧不再只是“补充说明”，而是这页的第二主区域。

选中一个市场后，右侧展示：

- 市场标题：城市 + 日期 + 温度区间
- 主价格区：`“是”价格`、`“否”价格`
- 盘口区：买一、卖一、价差
- 波动区：5 分钟变化、成交活跃程度、风险标签
- 状态区：更新时间、市场状态、是否关注
- 动作区：
  - 刷新当前筛选
  - 回到该城市
  - 切换精确明细模式

如果没有选中市场，则右侧显示“使用提示 + 当前筛选摘要”，而不是留空。

## 筛选区重构

筛选仍然保留，但表达方式升级为“两层结构”。

### 第一层：快速筛选

放在标题下方，一屏可见：

- 城市搜索
- 预测日期
- 仅关注盘口
- 排序字段
- 排序方向
- 视图模式切换

### 第二层：当前筛选摘要

用筛选标签集中展示当前状态，例如：

- 城市：东京
- 日期：2026-04-22
- 排序：5 分钟变化 / 降序
- 仅看关注盘口
- 当前结果：18 / 124

这样用户不用回头读输入框，也能一眼知道“页面现在处于什么筛选条件下”。

## 视图模式设计

第一阶段建议保留 2 个视图模式：

### 1. 总览模式

默认模式，采用市场带状总览。

适合：

- 快速扫城市
- 对比多个市场
- 找异常项
- 从首页跳转后继续深入

### 2. 精确模式

保留表格，但降为次级模式。

适合：

- 严格核对数值
- 批量复制
- 技术排查
- 高密度比对

这样不是简单“删表格”，而是把表格放回更合理的位置。

## 和首页的关系

首页泡泡页负责“先发现哪里有问题”。

市场总览页负责“进去之后快速判断到底是哪一个市场更值得看”。

所以两页需要连续：

- 首页双击城市泡泡进入后，市场总览自动带入城市和日期筛选。
- 市场总览页默认把该城市分组置顶。
- 右侧检视器的风格尽量沿用首页侧栏的语言和层级。

## 数据映射

基于现有 `MarketRow`，第一阶段可以直接做这些映射：

- `cityName`：城市分组标题
- `eventDate`：分组内辅助信息 / 全局筛选
- `temperatureBand`：市场带主标题
- `yesPrice`：主视觉数字
- `bestBid` / `bestAsk`：检视器明细
- `spread`：高价差强调
- `change5m`：趋势表达
- `status`：停牌 / 已结算状态标识
- `bubbleSeverity`：视觉风险主色
- `bubbleScore`：可作为排序辅助或密度提示
- `updatedAt`：新鲜度提示
- `watchlisted`：关注标识

## 可直接复用的现有能力

为了降低第二步实现成本，下面这些能力可以直接复用：

- 现有 `MarketQuery` 查询结构
- 现有 `MarketExplorerView` 的筛选状态同步逻辑
- 现有 `market-display` 里的中文格式化工具
- 现有 `monitor.css` 的表单、按钮、面板设计 token
- 现有首页中“右侧信息区”的深色玻璃质感语义

## 第一阶段不做的事

这一步先不扩任务边界，不做下面这些：

- 不先改后端接口
- 不先加复杂图表库
- 不先做多页签探索工作流
- 不先做拖拽编排
- 不先做导出和批量操作

先把“主视图从表格改成可视化工作台”做对。

## 第二步实施时的落地建议

进入实现时，优先顺序建议是：

1. 先把页面骨架从“单面板表格”改成“摘要 + 筛选 + 主视图 + 检视器”
2. 再把主视图改成市场带状总览
3. 最后再补精确模式和动画细节

这样每一步都能单独验收，不会再次变成过重任务。
